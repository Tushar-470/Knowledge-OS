## 4.1 EMA DRUG APPROVAL PROCESS

  

### Three Authorization Procedures

  

| Procedure | Full Name | Scope | Decision Made By | When Used |

|:---|:---|:---|:---|:---|

| **Centralised (CP)** | Centralised Procedure | Single application → valid in ALL EU/EEA member states | European Commission (based on EMA CHMP opinion) | **Mandatory** for: biotech, orphan drugs, HIV, cancer, neurodegenerative, diabetes, autoimmune. **Optional** for other innovative products |

| **Decentralised (DCP)** | Decentralised Procedure | Simultaneous application to multiple member states; one Reference Member State (RMS) leads | National authorities of each CMS | New products NOT yet approved in any EU state; when not required to use CP |

| **MRP** | Mutual Recognition Procedure | Product already approved in one member state → extend to others | National authorities recognize RMS decision | Existing products seeking multi-market access |

  

### Centralised Procedure Timeline

  

```

Pre-submission → Day 0: Application Filed → Day 1: Validation (starts clock)

    │

    ▼

Day 1–120: Primary Assessment by Rapporteur + Co-Rapporteur (CHMP members)

    │ → Preliminary Assessment Report issued

    ▼

Day 121: Clock Stop #1 → Applicant responds to questions (typically 3–6 months)

    │

    ▼

Clock restarts → Day 121–180: Evaluation of responses

    │

    ▼

Day 181: Clock Stop #2 (if needed) → Additional questions (oral explanation possible)

    │

    ▼

Day 210: CHMP Opinion (positive / negative / conditional)

    │

    ▼

Day ~280: European Commission Decision → Marketing Authorization granted

    │

    Total real time: ~12–15 months (with clock stops)

```

  

### Key EMA Bodies

  

| Body | Role |

|:---|:---|

| **CHMP** (Committee for Medicinal Products for Human Use) | Scientific assessment of medicines; issues opinions for CP |

| **PRAC** (Pharmacovigilance Risk Assessment Committee) | Safety monitoring; risk management plans |

| **CMDh** (Co-ordination Group for MRP/DCP) | Resolves disagreements in DCP/MRP procedures |

| **CAT** (Committee for Advanced Therapies) | ATMPs (gene therapy, cell therapy, tissue-engineered products) |

| **PDCO** (Paediatric Committee) | Paediatric Investigation Plans (PIP) |

  

### PRIME Scheme (PRIority MEdicines)

  

| Aspect | Detail |

|:---|:---|

| **What** | EMA support scheme for promising medicines addressing unmet medical needs |

| **Benefits** | Early EMA dialogue; assigned rapporteur; accelerated assessment possible (150 days vs 210) |

| **Eligibility** | Must demonstrate substantial advantage over existing treatments; supported by early clinical evidence |

| **Analogy** | Similar to FDA Breakthrough Therapy designation |

| **Annex 2** | Manufacture of Biological Active Substances and Products | Biologics manufacturing; biosimilar development |

| **Annex 3** | Manufacture of Radiopharmaceuticals | Specialized |

| **Annex 4** | Manufacture of Veterinary Medicinal Products (non-immunological) | Veterinary |

| **Annex 5** | Manufacture of Immunological Veterinary Medicinal Products | Veterinary |

| **Annex 6** | Manufacture of Medicinal Gases | Specialized |

| **Annex 7** | Manufacture of Herbal Medicinal Products | Herbal medicines |

| **Annex 8** | Sampling of Starting and Packaging Materials | Raw material sampling plans |

| **Annex 9** | Manufacture of Liquids, Creams and Ointments | Semi-solid manufacturing |

| **Annex 10** | Manufacture of Pressurised Metered Dose Aerosol Preparations | MDI manufacturing |

| **Annex 11** | **Computerised Systems** | **Data integrity; electronic records; equivalent to 21 CFR Part 11** |

| **Annex 12** | Use of Ionising Radiation in Manufacture | Sterilization |

| **Annex 13** | Manufacture of Investigational Medicinal Products (IMPs) | **Clinical trial supplies — R&D scientists encounter this directly** |

| **Annex 14** | Manufacture of Medicinal Products from Human Blood/Plasma | Blood products |

| **Annex 15** | **Qualification and Validation** | **IQ/OQ/PQ; process validation; cleaning validation; CSV** |

| **Annex 16** | Certification by a Qualified Person (QP) | QP batch release (EU-specific role) |

| **Annex 17** | Parametric Release | Real-time release without end-product testing |

| **Annex 18** | GMP for Active Pharmaceutical Ingredients (APIs) | Equivalent to ICH Q7 |

| **Annex 19** | Reference and Retention Samples | Sample retention requirements |

| **Annex 20** | Quality Risk Management | Implementation of ICH Q9 |

  

### Annex 1 (2023 Revision) — Key Points

  

| Topic | Requirement |

|:---|:---|

| **Contamination Control Strategy (CCS)** | **New requirement** — holistic, documented strategy covering all contamination risks (microbial, particulate, pyrogen, cross-contamination) |

| **Cleanroom Classification** | Grade A (critical zone, ISO 5); Grade B (background for aseptic); Grade C (less critical steps); Grade D (least critical) |

| **APS (Aseptic Process Simulation)** | Must simulate worst-case production conditions; minimum 3 successful consecutive runs; ≥5000 units per run |

| **Barrier Technology** | RABS (Restricted Access Barrier System) and isolators preferred over conventional cleanrooms |

| **Environmental Monitoring** | Continuous viable + non-viable particle monitoring in Grade A zones |

| **Personnel** | Gowning qualification; regular requalification; behavior-based training |

  

### Annex 11 — Computerised Systems (Data Integrity)

  

| Requirement | Detail |

|:---|:---|

| **Risk assessment** | All computerised systems must have documented risk assessment |

| **Validation** | System must be validated before use; IQ/OQ/PQ documented |

| **Audit trail** | All GMP-relevant changes recorded; audit trail review mandatory |

| **Electronic signatures** | Must be linked to individual user; unique IDs; no shared logins |

| **Data backup** | Regular, verified backups; disaster recovery plan |

| **Access control** | Role-based access; admin rights restricted; periodic user access review |

| **Printouts** | If electronic system fails, printout must be available as backup |

## 4.4 EMA vs FDA — KEY DIFFERENCES

  

| Aspect | FDA (USA) | EMA (EU) |

|:---|:---|:---|

| **Legal basis** | Federal Food, Drug, & Cosmetic Act (FD&C Act) | Regulation (EC) No 726/2004; Directive 2001/83/EC |

| **Single authority?** | Yes — FDA is the sole regulator | No — EMA coordinates; national agencies execute |

| **GMP standard** | 21 CFR 210/211 | EU GMP Guide (Parts I, II, Annexes 1–20) |

| **Data integrity** | 21 CFR Part 11 | EU GMP Annex 11 |

| **Stability (Zone)** | Zone II (25°C/60%RH) | Zone II (25°C/60%RH) |

| **Batch release** | QC department releases | **Qualified Person (QP)** must certify each batch — legally mandated individual |

| **Pharmacopoeia** | USP | PhEur (European Pharmacopoeia) |

| **Generics pathway** | ANDA (505(j)) | Abridged application (Article 10) referencing innovator dossier |

| **Biologics pathway** | BLA (Biologics License Application) | Centralised Procedure (mandatory) |

| **Biosimilar standard** | 351(k) BLA | Centralised + biosimilar guideline |

| **Inspections** | FDA conducts globally (including Indian sites) | National inspectors (coordinated by EMA) |

| **Naming** | Brand name + nonproprietary name | INN (International Nonproprietary Name) |

| **OTC classification** | Extensive OTC drug category (monograph system) | More restrictive; national differences |

| **Paediatric** | PREA (Pediatric Research Equity Act) | PIP (Paediatric Investigation Plan) — mandatory |

| **Orphan drugs** | 7-year market exclusivity | 10-year market exclusivity |

| **Post-approval changes** | CBE-0, CBE-30, PAS (supplement-based) | Variations (Type IA, IB, II) |

## 5.1 GLP (Good Laboratory Practice)

  

| Aspect | Detail |

|:---|:---|

| **What** | Quality system for non-clinical safety studies (toxicology, genotoxicity, PK in animals) |

| **Regulatory basis** | OECD Principles of GLP; 21 CFR Part 58 (FDA); Schedule L1 (India) |

| **Who follows** | Preclinical research labs; CROs conducting tox studies |

| **R&D Scientist relevance** | If your formulation goes to animal tox → the tox study must be GLP; you provide test article characterization data |

  

**Key GLP Requirements:**

  

| Requirement | What It Means |

|:---|:---|

| **Study Director** | One individual responsible for overall conduct of study |

| **Study Plan (Protocol)** | Written before study begins; amendments documented |

| **Quality Assurance Unit** | Independent QA inspects study conduct and data |

| **Standard Operating Procedures** | All procedures documented |

| **Test and Reference Articles** | Characterized (identity, purity, stability); chain of custody documented |

| **Raw Data** | All observations recorded promptly; corrections dated, initialed, reason given |

| **Archive** | All data, protocols, reports archived for defined retention period |

| **Final Report** | Signed by Study Director; includes all raw data references |

## 5.2 GMP IN R&D — WHAT YOU MUST FOLLOW

  

> [!WARNING]

> **Common misconception:** "GMP doesn't apply to R&D." This is WRONG. While full cGMP applies to commercial manufacturing, R&D scientists must follow **GMP-appropriate practices** to ensure data integrity, traceability, and successful technology transfer.

  

### GMP Practices Required in R&D Labs

  

| Practice | Why |

|:---|:---|

| **Lab notebooks with ink entries** | Contemporaneous recording; no pencil; no correction fluid |

| **Proper error corrections** | Single line through error; initial + date + reason; original visible |

| **Equipment calibration** | Balances, pH meters, HPLC — must be calibrated before use |

| **Reference standard management** | CoA on file; proper storage; expiry tracking |

| **SOP compliance** | All procedures documented; training records maintained |

| **Data integrity** | No data deletion; all chromatographic processing changes documented |

| **Sample chain of custody** | Know where every sample came from and where it went |

| **Proper labeling** | All solutions, standards, samples labeled with content, date, analyst initials |

| **Development batch records** | Even trial batches should have records — they become part of CTD P.2 |

| **Change control** | Any change to method or formula during development → document rationale |

## 5.5 DATA INTEGRITY — ALCOA+ PRINCIPLES

  

| Principle | Meaning | Example in R&D Lab |

|:---|:---|:---|

| **A**ttributable | WHO did it? | Each entry signed/initialed; login ID on electronic systems; analyst name on chromatogram |

| **L**egible | Can you READ it? | Clear handwriting in lab notebooks; printouts attached; electronic records readable |

| **C**ontemporaneous | Recorded at the TIME of activity | Record results immediately; no "catching up" lab notebooks at end of day |

| **O**riginal | FIRST recording | Original chromatogram = electronic file in CDS (not printout); raw data is the original |

| **A**ccurate | Correct and TRUTHFUL | No fabrication; no selective reporting; calculations verified |

| + **C**omplete | ALL data included | Report ALL results including OOS; no trial injections hidden; no selective deletion |

| + **C**onsistent | No contradictions | Logbook time matches instrument audit trail time; batch record matches lab notebook |

| + **E**nduring | PERMANENT | Use non-erasable ink; electronic records backed up; no sticky notes as permanent records |

| + **A**vailable | Retrievable when needed | Archived data accessible; backup systems functional; retention period met |

  

### Data Integrity Red Flags (What Auditors Look For)

  

| Red Flag | What It Indicates |

|:---|:---|

| Multiple trial injections before "final" analysis | Possible cherry-picking of best result |

| Audit trail disabled or not reviewed | System not compliant; data may be altered |

| Shared login credentials | Cannot attribute actions to individuals |

| Files deleted from CDS recycle bin | Data may have been hidden |

| Lab notebook entries all in same ink/handwriting for entire day | May not be contemporaneous |

| Unusual working hours (2 AM test runs) | Possible unauthorized testing |

| Date/time mismatch between equipment log and data file | Backdating suspected |

| Only passing results in files; no failures documented | Selective reporting |

|:---|:---|

## 6.1 CAPA — CORRECTIVE AND PREVENTIVE ACTION

  

### 10-Step CAPA Process

  

```

STEP 1: IDENTIFICATION

   → Source: OOS result, deviation, complaint, audit finding, trend analysis

   → Enter into CAPA tracking system with unique number

       ↓

STEP 2: EVALUATION

   → Is CAPA required? (Not every issue needs full CAPA)

   → Risk assessment: What is the impact on product quality / patient safety?

   → Assign CAPA owner and target completion date

       ↓

STEP 3: ROOT CAUSE INVESTIGATION

   → Use tools: 5 Whys, Fishbone (Ishikawa), Fault Tree Analysis

   → Identify TRUE root cause, not just symptoms

   → Example: "Why did tablet hardness fail?" → "Granulation moisture was 4.5%

     instead of 3.0%" → "Endpoint moisture was not checked" → "LOD sensor

     was out of calibration" → ROOT CAUSE: Calibration schedule not followed

       ↓

STEP 4: CORRECTIVE ACTION (Fix the Current Problem)

   → Immediate action to address the specific incident

   → Example: Recalibrate sensor; reprocess or reject affected batch

       ↓

STEP 5: PREVENTIVE ACTION (Prevent Recurrence)

   → Systemic change to prevent same issue in future

   → Example: Implement automated calibration reminder system; add pre-use

     calibration check to SOP; retrain all operators

       ↓

STEP 6: IMPLEMENTATION PLAN

   → Detailed action items with responsible persons and deadlines

       ↓

STEP 7: APPROVAL

   → QA reviews and approves CAPA plan

       ↓

STEP 8: EXECUTION

   → Implement all corrective and preventive actions per plan

       ↓

STEP 9: EFFECTIVENESS CHECK

   → After defined period (30, 60, 90 days), verify that the problem has

     not recurred and the preventive actions are working

   → This is the step MOST COMMONLY missed — major audit finding

       ↓

STEP 10: CLOSURE

   → QA reviews effectiveness data and closes CAPA

   → Document in annual product quality review (APQR)

```

## 6.2 CHANGE CONTROL SYSTEM

  

| Stage | Detail |

|:---|:---|

| **Initiation** | Any change to approved process, equipment, material, method, specification, or facility |

| **Classification** | **Minor:** No impact on product quality (e.g., cosmetic label change); **Major:** Potential impact on quality (e.g., API supplier change, process parameter change, new equipment) |

| **Impact assessment** | Cross-functional review: R&D, QA, QC, Production, Regulatory, Supply Chain |

| **Regulatory assessment** | Does this change require a regulatory filing? (CBE-30? PAS? Type II variation?) |

| **Approval** | QA approval mandatory; Management approval for major changes |

| **Implementation** | Execute change per plan; train affected personnel |

| **Post-implementation review** | Verify change has desired effect; no adverse consequences |

| **Documentation** | Complete change control record filed |

  

### Change Control Examples Relevant to R&D

  

| Change | Classification | Regulatory Impact |

|:---|:---|:---|

| API supplier change (same spec) | Major | ANDA: CBE-30 supplement; EU: Type II variation |

| Change HPLC column brand (same dimensions) | Minor | None if SST still passes; document in method lifecycle |

| Change excipient grade (e.g., MCC PH-102 → PH-101) | Major | May require dissolution comparison + stability data |

| Change dissolution test medium | Major | Regulatory filing required (new specification) |

| Change stability chamber | Minor | Requalify new chamber; no filing |

| Change manufacturing site | Major | Prior Approval Supplement (FDA); Type II variation (EU) |

## 6.3 DEVIATION MANAGEMENT

  

### Deviation Classification

  

| Type | Definition | Example | Action |

|:---|:---|:---|:---|

| **Planned Deviation** | Pre-approved departure from SOP (temporary) | Using alternative glassware because volumetric flask broken | Pre-approve; document; assess impact |

| **Unplanned Minor** | Unexpected; no impact on product quality | Room temperature briefly exceeded limit but product not exposed | Document; investigate; close |

| **Unplanned Major** | Unexpected; potential impact on quality | Wrong excipient weighed; equipment malfunction during coating | Full investigation; CAPA; may require batch rejection |

| **Critical** | Significant impact on product quality or patient safety | Cross-contamination detected; sterility breach | Immediate investigation; batch quarantine; regulatory notification may be required |

## 6.4 OOS / OOT INVESTIGATION

  

### OOS (Out of Specification) Investigation Flowchart

  

```

OOS RESULT OBTAINED

        │

        ▼

PHASE 1: LAB INVESTIGATION (within 3 working days)

   → Analyst reviews:

     ✓ Calculations correct?

     ✓ Correct reference standard used?

     ✓ Sample preparation correct (dilutions, volumes)?

     ✓ System suitability passed?

     ✓ Instrument functioning properly (check log)?

     ✓ Chromatogram normal (no anomalies)?

     ✓ Correct method/specification applied?

        │

        ├─── ASSIGNABLE LAB ERROR FOUND

        │    → Document error; invalidate result with justification

        │    → Re-prepare and re-test (NOT retesting same preparation)

        │    → Close Phase 1

        │

        └─── NO LAB ERROR FOUND → Escalate to Phase 2

                │

                ▼

PHASE 2: EXTENDED INVESTIGATION (within 30 working days)

   → QA involvement mandatory

   → Re-sample from same batch (new sample)

   → Test by second analyst on second instrument (if possible)

   → Review:

     ✓ Batch manufacturing record for deviations

     ✓ Environmental monitoring data

     ✓ Equipment maintenance records

     ✓ Raw material CoAs

     ✓ Previous batch results (trending)

        │

        ├─── ROOT CAUSE IDENTIFIED

        │    → Document root cause

        │    → Initiate CAPA

        │    → Decide: batch release / reject / rework

        │

        └─── ROOT CAUSE NOT IDENTIFIED

             → Batch cannot be released

             → Document as "inconclusive — root cause not determined"

             → Initiate preventive CAPA

             → Consider additional testing, third-party testing

```

  

### OOT (Out of Trend) — Different from OOS

  

| Aspect | OOS | OOT |

|:---|:---|:---|

| **Definition** | Result outside specification limits | Result within spec but showing unexpected trend |

| **Example** | Assay = 88.5% (spec: 90–110%) | Assay declining: 99.2% → 97.8% → 96.1% → 94.5% over 4 timepoints |

| **Action** | Formal OOS investigation required | Statistical trend analysis; may trigger accelerated stability or investigation |

| **Regulatory concern** | Immediate impact on batch disposition | Predictive; may indicate future OOS if trend continues |

## 6.5 RISK ASSESSMENT TOOLS

  

### FMEA (Failure Mode and Effects Analysis)

  

**Used for:** Process risk assessment; CQA identification; formulation optimization

  

| Column | Description |

|:---|:---|

| **Process Step** | Each unit operation |

| **Failure Mode** | What can go wrong? |

| **Effect** | What is the impact on product quality? |

| **Severity (S)** | 1–10 scale (10 = most severe) |

| **Occurrence (O)** | 1–10 scale (10 = most likely) |

| **Detection (D)** | 1–10 scale (10 = least detectable) |

| **RPN** | Risk Priority Number = S × O × D (max = 1000) |

| **Action** | If RPN > threshold (e.g., >100), take action to reduce risk |

  

**Worked Example — Tablet Compression:**

  

| Failure Mode | Effect | S | O | D | RPN | Action |

|:---|:---|:--|:--|:--|:----|:---|

| Incorrect compression force | Hardness OOS → dissolution failure | 8 | 3 | 4 | 96 | Monitor; in-process check every 30 min |

| Weight variation | Content uniformity failure | 9 | 4 | 3 | 108 | **Action needed:** Auto-reject system; weight check every 15 min |

| Punch sticking | Picking/pitting defects; weight variation | 6 | 5 | 5 | 150 | **Action needed:** Anti-adherent in formula; polish punches; reduce turret speed |

| Cross-contamination | Wrong drug in product → patient harm | 10 | 2 | 6 | 120 | **Action needed:** Validated line clearance; cleaning validation |

  

### HACCP (Hazard Analysis Critical Control Points)

- Originally for food safety; adapted for pharma

- Identifies Critical Control Points (CCPs) where monitoring is essential

- Used for: sterile manufacturing contamination control; water system monitoring

  

### Fishbone / Ishikawa Diagram

- **Categories (6M):** Man, Machine, Method, Material, Measurement, Mother Nature (Environment)

- Used for root cause investigation during CAPA and OOS

- Draw the "bone" for each category → brainstorm possible causes → investigate most likely

## 7A) INDIAN PHARMA COMPANY INTERVIEWS (15 Questions)

  

**Q1: "What is Schedule M and why is it important?"**

> Schedule M of the Drugs and Cosmetics Rules, 1945 defines the GMP requirements for pharmaceutical manufacturing in India. It covers premises, equipment, personnel, documentation, production, quality control, self-inspections, and complaints. Revised Schedule M (2024) brought Indian GMP closer to WHO standards. Compliance is mandatory for all drug manufacturing licenses. Non-compliance can lead to license suspension or cancellation by the State Drug Controller.

  

**Q2: "What is the SUGAM portal?"**

> SUGAM is CDSCO's online platform for all regulatory applications — new drug approvals, clinical trial applications, import licenses, manufacturing licenses. Since 2019, all applications must be submitted electronically via SUGAM. It provides real-time tracking of application status and has significantly improved transparency and reduced processing times.

  

**Q3: "What are the stability conditions for Zone IVa?"**

> India falls under ICH Climatic Zone IVa. Long-term stability: 30°C ± 2°C / 75% RH ± 5% for minimum 12 months. Accelerated: 40°C ± 2°C / 75% RH ± 5% for 6 months. This is more stringent than Zone II (USA/EU: 25°C/60%RH) because of India's hot and humid climate. Products marketed in India must be tested under Zone IVa conditions.

  

**Q4: "Explain the difference between a new drug and an existing drug under Indian regulations."**

> A "new drug" per NDCT Rules 2019 includes: an NCE; a new salt/ester of approved drug; a fixed-dose combination not previously marketed; a modified-release form of approved drug; a drug approved for new indication; any drug NOT approved in India for >4 years. An "existing drug" is one that has been approved and marketed in India for >4 years and is included in Indian Pharmacopoeia. New drugs are approved by CDSCO; existing drugs can be licensed by State Drug Controllers.

  

**Q5: "What is the role of the Subject Expert Committee (SEC)?"**

> The SEC is a panel of domain experts appointed by CDSCO to evaluate applications for new drugs, clinical trials, and biologics. They review scientific data (preclinical, clinical, CMC) and provide recommendations to the DCGI. SEC meetings are drug category-specific (e.g., separate SEC for pulmonary, oncology, cardiovascular). Their recommendation is advisory — final approval authority rests with DCGI.

  

**Q6: "How does the Indian Pharmacopoeia (IP) differ from USP?"**

> Both define standards for drug quality. IP uses BAN/INN nomenclature while USP uses USAN. IP specifies Zone IVa stability conditions; USP specifies Zone II. Some monograph tests and acceptance criteria differ. For domestic Indian market, IP is the legally binding reference. For US exports, USP compliance is mandatory. Companies typically set internal specs meeting both.

  

**Q7: "What is the bioequivalence requirement for generic drugs in India?"**

> For oral solid dosage forms, CDSCO requires a bioequivalence study demonstrating that the generic has the same rate and extent of absorption as the reference product. Study design: randomized, crossover, 2-period, in healthy volunteers. 90% CI for Cmax and AUC must fall within 80.00–125.00%. BCS-based biowaivers are accepted for Class I drugs with rapid dissolution.

## 7B) US PHARMA COMPANY INTERVIEWS (15 Questions)

  

**Q1: "Explain the ANDA filing process for a generic drug."**

> Covered in detail in Part 3 of regulatory_gxp_part1.md. Key steps: select RLD from Orange Book → develop pharma equivalent → conduct BE study → file ANDA in CTD format → Paragraph IV certification if patents exist → FDA OGD review → PAI → approval. For an R&D scientist, the contribution is primarily to Module 3 (Quality): pharmaceutical development (P.2), analytical methods (P.5), and stability (P.8).

  

**Q2: "What is 21 CFR Part 11 and how does it affect your daily work?"**

> Part 11 governs electronic records and electronic signatures. In my daily work: (1) I use unique login credentials for Empower CDS — no shared passwords. (2) All HPLC data processing changes are captured in the audit trail. (3) I cannot delete raw data files. (4) My electronic signature on reports is legally binding. (5) The system automatically locks after inactivity. Understanding Part 11 is critical because FDA 483 observations for data integrity are the #1 category of findings.

  

**Q3: "What is a design space and what advantage does it give in regulatory filing?"**

> Design space (ICH Q8) is the multidimensional combination of input variables and process parameters demonstrated to provide quality assurance. Advantage: Once approved in the filing, changes WITHIN the design space do NOT require prior FDA approval (no supplement needed). Changes OUTSIDE the design space require a post-approval supplement. This gives manufacturing flexibility — e.g., if your design space for granulation moisture is 2–4%, you can operate anywhere in that range without regulatory paperwork.

  

**Q4: "Describe a situation where you had to investigate an unexpected lab result."**

> **STAR format (Situation-Task-Action-Result):** "(Situation) During stability testing of my SR tablet at 3-month accelerated, dissolution at 8h was 68% instead of expected >80%. (Task) I needed to determine if this was a true stability change or analytical error. (Action) Phase 1: I verified calculations, sample prep, system suitability — all correct. I checked dissolution apparatus calibration — within spec. Phase 2: I re-sampled from the stability chamber, ran on a different dissolution apparatus. Result was again 67%. I reviewed the batch record — the coating level for this particular stability batch was at the lower end of the range. I conducted forced degradation to check if degradants were affecting release. (Result) Root cause: insufficient coating at lower spray rate → premature matrix erosion at accelerated conditions. I recommended optimizing the coating weight gain from 4% to 6% and initiated CAPA to tighten the coating process parameter."

  

**Q5: "What is the difference between a CBE-0, CBE-30, and PAS?"**

> These are post-approval supplement types for reporting changes to approved NDAs/ANDAs. **CBE-0 (Changes Being Effected):** Minor changes that can be implemented IMMEDIATELY upon filing. Example: tightening a specification. **CBE-30:** Moderate changes; must wait 30 days after filing for FDA to review before implementing. Example: change in analytical method. **PAS (Prior Approval Supplement):** Major changes; CANNOT implement until FDA approves. Example: change in API supplier, manufacturing site, or drug substance synthesis route. Classification follows SUPAC guidance (Scale-Up and Post-Approval Changes).

  

**Q6: "What are FDA Product-Specific Guidances (PSGs) and how do you use them?"**

> PSGs are FDA's recommendations for specific generic drug products. They tell you: (1) what the recommended BE study design is (fasting only? fasting + fed?); (2) which dissolution conditions to use; (3) whether BCS biowaiver is applicable; (4) specific formulation considerations. Found on FDA's website under OGD. You use them BEFORE starting development to align your strategy with FDA expectations. Example: PSG for Metformin HCl ER tablets specifies USP Apparatus I (basket), 100 RPM, specific media, and recommends a fed BE study.

## 7C) EU PHARMA COMPANY INTERVIEWS (15 Questions)

  

**Q1: "What is the role of the Qualified Person (QP) in EU?"**

> The QP is a legally mandated individual responsible for certifying that each batch of medicinal product has been manufactured and checked in accordance with EU GMP, the marketing authorization, and applicable regulations. No batch can be released to market without QP certification (EU GMP Annex 16). The QP must hold a degree in pharmacy, chemistry, biology, or medicine and have ≥2 years practical experience. This role has NO equivalent in the US system — in the US, QC releases batches.

  

**Q2: "Explain the difference between Centralised, Decentralised, and Mutual Recognition procedures."**

> Covered in Part 4.1. Centralised = one application to EMA → valid in all 27 EU/EEA states. Mandatory for biotech, orphan, oncology, etc. Decentralised = simultaneous application to multiple states with one RMS; for products not yet approved anywhere in EU. MRP = extend existing national approval to other member states. Choose based on product type and commercial strategy.

  

**Q3: "What is EU GMP Annex 1 and what changed in the 2023 revision?"**

> Annex 1 covers manufacture of sterile medicinal products. The 2023 revision (effective August 2023) is the first major update in decades. Key changes: (1) Introduction of **Contamination Control Strategy (CCS)** — a holistic, documented strategy. (2) Emphasis on barrier technology (RABS/isolators) over conventional cleanrooms. (3) Enhanced environmental monitoring requirements. (4) APS requirements strengthened (≥5000 units per simulation). (5) Expanded scope to include non-sterile products requiring aseptic processing.

  

**Q4: "What is EU Annex 11 and how does it compare to 21 CFR Part 11?"**

> Both govern computerised systems and data integrity. Annex 11 is the EU equivalent of Part 11. Key similarities: audit trail, access control, validation, electronic signatures. Key differences: Annex 11 is more prescriptive about risk assessment of computerised systems; requires a nominated "process owner" for each system; emphasizes data migration validation when upgrading systems. Both require that electronic records are as reliable as paper records.

  

**Q5: "What is the European Pharmacopoeia (PhEur) and how does it differ from USP?"**

> PhEur is published by EDQM (European Directorate for the Quality of Medicines). It is legally binding in all 39 signatory states. Key differences from USP: (1) PhEur uses European numbering for general chapters (e.g., 2.2.29 for liquid chromatography vs USP <621>). (2) Some test methods differ (e.g., microbial limits, dissolution specifications). (3) Naming uses INN exclusively. (4) PhEur monographs may have different impurity limits. (5) CEP (Certificate of Suitability) system for API sourcing — unique to Europe.

  

**Q6: "What is a CEP (Certificate of Suitability) and why is it important?"**

> CEP is issued by EDQM when an API manufacturer demonstrates that their API meets the quality requirements of the relevant PhEur monograph. The CEP covers manufacturing process, impurity profile, specifications, and stability. Benefits: (1) Simplifies regulatory filings — the drug product applicant references the CEP instead of disclosing full API details. (2) Protects API manufacturer's IP. (3) Mutual recognition across all EU member states. (4) Widely accepted beyond EU (Canada, Australia, many ASEAN countries). Indian API companies (DRL, Aurobindo, Laurus Labs) routinely obtain CEPs for their APIs.
